# Governance Migration Plan

Iteration: `baseline-v0.1`

Status: `draft`

Artifact: `ART-026`

## Migration Strategy

The baseline path is incremental and lawful: build compatible civic infrastructure inside existing jurisdictions, then federate successful pilots. The plan avoids assuming a blank-slate state.

## Phases

| Phase | Description | Legal Form Options |
| --- | --- | --- |
| MIG-001 | Research and charter formation | Nonprofit project, academic consortium, public-benefit entity. |
| MIG-002 | Service pilot | Municipal partnership, cooperative, community land trust, clinic/school partnership. |
| MIG-003 | Multi-community federation | Interlocal agreement, cooperative federation, public authority, foundation-backed network. |
| MIG-004 | Formal public integration | Charter amendment, municipal ordinance, statutory authority, public trust. |
| MIG-005 | Constitutional scaling | Requires democratic mandate and compatibility with existing constitutional law. |

## Legal Compatibility Checks

- Civil rights and non-discrimination.
- Data protection and privacy.
- Labor and employment rules.
- Public procurement and anti-corruption law.
- Health, education, housing, and emergency management regulation.
- Administrative procedure and judicial review.

## Migration Gates

| Gate ID | Gate | Required Evidence |
| --- | --- | --- |
| GATE-001 | Service reliability | Service-level metrics meet published target. |
| GATE-002 | Rights safety | No unresolved systemic rights violations. |
| GATE-003 | Financial viability | Maintenance and reserve funding demonstrated. |
| GATE-004 | Public legitimacy | Participation, dissent handling, and appeal outcomes reviewed. |
| GATE-005 | Auditability | Decision, procurement, data, and risk logs functioning. |

## Rollback

Any phase can be sunset if rights violations, insolvency, hidden capture, data misuse, or service failure cannot be corrected within a defined remediation period.
