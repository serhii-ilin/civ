# Adoption Playbook

Iteration: `baseline-v0.1`

Status: `draft`

Artifact: `ART-025`

## Adoption Principles

- Start with voluntary, legally compatible pilots.
- Build trust by improving concrete services before expanding authority.
- Publish decisions, budgets, metrics, and mistakes.
- Keep personal data private by default.
- Use existing institutions where possible instead of requiring rupture.

## Onboarding Sequence

1. Publish the charter, rights floor, data policy, and dispute pathways.
2. Hold community deliberation with accessible materials and dissent capture.
3. Map existing services, gaps, risks, and trusted local actors.
4. Create a first-year service plan with measurable guarantees.
5. Elect or select initial oversight bodies under conflict rules.
6. Start public decision log and risk register.
7. Run a tabletop stress scenario before taking on critical functions.

## Community Practices

- Regular public assemblies with opt-out and asynchronous participation.
- Care network enrollment by consent.
- Newcomer orientation and rights education.
- Voluntary civic rituals focused on service, repair, grief, and celebration.
- Translation and accessibility support.

## Adoption Risks

| Risk | Mitigation |
| --- | --- |
| Movement capture by charismatic founders | Term limits, shared control, independent audit, public finances. |
| Ideological sorting | Pluralism rules, rights floor, broad service focus, anti-discrimination enforcement. |
| Privacy backlash | Data minimization, opt-in care checks, public institutional logs only. |
| Service overpromising | Service-level guarantees tied to funding and capacity. |
| Local opposition | Legal compatibility, transparent scope, grievance process. |
